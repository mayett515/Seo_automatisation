# Planner Questions

Use these to interrogate a new schema target.

## Project identity

1. What is the target stack?
2. Is this a monorepo, single app, library, or learning workspace?
3. What is the primary purpose of the schema?

## Architecture

1. What folder structure does the codebase use?
2. What architectural style should be enforced?
3. What should be banned?

## Tooling

1. What package manager?
2. What test commands?
3. What build commands?
4. What lint/format commands?

## Agent behavior

1. What tasks should agents do often?
2. What tasks should agents never do?
3. Should there be skills?
4. Should there be subagents?
5. Should there be hooks?

## Context sharding

1. Are there separate cognitive modes?
2. Are there frontend/backend/devops/refactor/planner modes?
3. Should each have its own hidden shard?
