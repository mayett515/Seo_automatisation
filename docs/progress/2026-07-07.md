## Page Studio CSS And Theme Direction

- Folded the later Big Eater Page Studio and CSS pattern-mining findings into ADR 0017, the roadmap, and the frontend/Page Registry architecture doc.
- Recorded the customer-page styling boundary:
  - the operator app remains the React/TanStack control surface;
  - customer pages are deterministic static HTML/CSS emitted from PageJson and code-owned registry entries;
  - the renderer owns class names, CSS custom properties, data attributes, layout primitives, and section CSS;
  - PageJson stores structured style choices only, such as variant, zone, density, media placement, CTA emphasis, proof visibility, and future theme preset identifiers.

- Recorded the first CSS foundation direction for renderer migration:
  - semantic CSS custom properties;
  - cascade layers;
  - small layout primitives such as stack, cluster, container, grid, sidebar, frame, and bleed;
  - renderer-owned data attributes for validated variants and states.
- Kept `packages/page-registry` as the initial home for CSS/theme code. A separate `packages/page-theme` remains deferred until static deploy rendering, preview rendering, and Page Studio share enough duplicated theme APIs to justify the package boundary.
