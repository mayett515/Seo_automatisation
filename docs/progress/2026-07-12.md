## Project-Scoped Media Asset Architecture

- Ran a focused Good Artist Inspiration pass across the local multi-driver storage catalog, AWS presigned POST and object-metadata guidance, OWASP upload controls, and Sharp decoder/output safety behavior.
- Accepted ADR 0020 before introducing media code or controls.
- Made Postgres the owner of project-scoped asset identity, lifecycle, actor evidence, recovery, and immutable derivative metadata; private object storage owns bytes.
- Fixed the PageJson boundary to opaque placement references with content/decorative alt semantics and an optional normalized focal point. Raw URLs, object keys, MIME types, filenames, and provider fields remain forbidden.
- Chose short-lived, exact-key, size/checksum-bound private quarantine upload followed by explicit completion and deterministic BullMQ processing; storage and queue availability plus project quotas are checked before accepting an intent.
- Limited MVP input to static JPEG, PNG, and WebP, with decoder byte/pixel limits, decoded-content validation, metadata stripping, and deterministic responsive WebP derivatives.
- Required ready/archived asset resolution at version/approval/release boundaries and a `page_version_media_assets` reference projection for retention safety. Ready manifests, processor versions, and derivative bytes are immutable; changed processing creates a new asset id.
- Required preview and deploy to use the same resolved manifest, renderer, HTML paths, and derivative bytes. Media therefore also requires an authenticated preview-document route and a binary-safe static artifact contract.
- Resolved the preview auth/sandbox/parity triangle with a five-minute path-scoped signed capability cookie while preserving `sandbox=""` and token-free rendered paths.
- Required Netlify digests and uploads to consume the same decoded file bytes, and required a DB-checked exact derivative set before an asset can become ready.
- Split implementation into backend ingestion/processing, renderer/deploy parity, and Page Studio controls so upload security is not mixed with visual editing.
