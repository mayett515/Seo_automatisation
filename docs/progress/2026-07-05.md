# Progress 2026-07-05

## SERP Snapshot Boundary

- Started the next search/SERP slice as a boundary and storage baseline, not live automation.
- Added normalized SERP contracts in `packages/contracts`:
  - `SerpScoutRequest`,
  - `SerpSnapshot`,
  - `SerpSearchResult`,
  - `SerpFeature`,
  - `SerpEngineError`,
  - `SerpArtifactRef`,
  - `SerpScoutFailureCode`.
- Added `SerpScoutPort.search(...)` in `packages/adapters` as the purpose-named read-only search boundary.
- Reserved read-only reasoning tool categories for future search/page tools:
  - `search_web`,
  - `read_public_page`.
- Added `serp_snapshots` persistence in `packages/db`:
  - project-owned source rows,
  - optional `agent_run_id` provenance,
  - query/search engine/device/locale/region/cache key,
  - normalized result, SERP feature, engine error, and artifact-ref JSON,
  - `captured_at` for proof freshness policy later.

## Design Clarification

- DeepSeek can drive the future SERP/search workflow:
  - plan queries,
  - choose nearby Orte/service combinations,
  - call read-only SERP tools,
  - interpret the results.
- DeepSeek does not create ranking truth by itself. The evidence boundary is the stored `serp_snapshots` row or artifact.
- CAPTCHA/anti-bot is modeled as `captcha_blocked`, a stop signal rather than something to bypass.
