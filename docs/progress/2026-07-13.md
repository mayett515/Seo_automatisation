# Progress - 2026-07-13

## Media Renderer, Preview, And Deploy Parity

- Completed ADR 0020 slice 2 without adding PageJson media fields or Page Studio media controls.
- Migrated `StaticSiteFile` to an explicit UTF-8/base64 discriminated contract with safe unique paths and a 50 MiB decoded artifact budget.
- Updated the Netlify adapter to decode each file once, calculate SHA1 over decoded bytes, and upload those exact bytes rather than base64 transport text.
- Added one project-scoped media-manifest resolver over the page-version reference projection; preview and deploy use the same ready/archived exact-variant truth.
- Replaced preview `srcDoc` transport with a metadata-only response and capability-authorized preview document while preserving `sandbox=""`.
- Added the two-stage preview capability chain: the guarded metadata endpoint issues a page-version document capability, and the document endpoint rechecks the manifest before issuing the `/assets` capability.
- Kept capability tokens out of rendered HTML and asset paths. Capability cookies are `HttpOnly`, `Secure`, `SameSite=None`, `Partitioned`, manifest-bound, and limited to five minutes in every environment; local Vite proxies `/api` and `/assets` through the same origin without falling back to `SameSite=Lax` for opaque-origin iframe requests.
- Added capability-only preview asset delivery that requires exact manifest membership and verifies object byte count plus SHA-256 before responding.
- Extended deploy artifact construction to read projected immutable derivative bytes, verify them, and embed base64 media files alongside UTF-8 HTML before object-storage persistence or provider handoff.
- Added contract, adapter, capability, API Postgres, worker unit/Postgres, and Chromium coverage for the binary and preview/deploy parity seams.
- Added a real-network Chromium test proving the document cookie reaches the sandboxed navigation and the `/assets` cookie reaches an image request from the iframe's opaque origin.
- Updated ADR 0017/0020, roadmap/status/frontend/Page Studio architecture docs, lifecycle coverage, Rule 15, and executable guards in the same slice.
