# 2026-06-29 Progress

## Scope

Backend foundation hardening after architecture/pattern-mining reviews, ending with the deterministic deploy reconciler baseline.

## Completed

- Added per-project publishable tracking keys with hashed storage, origin allowlists, create/list/revoke management, and public `/track` ingestion.
- Hardened `/track` with IP, IP/project, true project, key, and key/project rate-limit buckets.
- Coalesced tracking-key `lastUsedAt` writes so accepted events do not update the key row on every request.
- Replaced synthetic release preflight with persisted evidence-backed checks for approval, noindex, local SEO blockers/warnings, rollback readiness, and tracking readiness.
- Made release plan creation transactional and reject missing/cross-project page versions instead of silently dropping them.
- Made deploy approval transactional and state-machine guarded.
- Made deploy execution require a persisted approval row in addition to passing preflight checks.
- Split worker routing from GSC sync implementation and kept `job_runs` lifecycle updates in shared worker helpers.
- Moved `queueNames` to `@localseo/contracts` so API and worker share queue-name truth.
- Added `deployment_key`, deployment evidence JSON, and `release_verification_checks` schema support.
- Made migration `0009_lying_namor` backfill-safe by adding `deployment_key` nullable, filling existing rows, then enforcing `NOT NULL`.
- Expanded `SiteHostingPort` to provider-neutral create/read/restore/rollback methods.
- Changed `NotConfiguredSiteHostingAdapter` to return `not_configured` instead of fake deploy success.
- Implemented the deterministic deploy reconciler worker baseline:
  - parses shared `DeployJobDataSchema`,
  - derives a stable `deploymentKey`,
  - reloads release plan, checks, approval, rollback, and item evidence,
  - writes or reuses a deployment ledger row by `deployment_key`,
  - calls `SiteHostingPort.createDeploy`,
  - marks deployment/release as `provider_succeeded`/`live` on provider success,
  - marks deployment/release as `failed` on provider/configuration failure,
  - replays already-successful deployment rows without calling the provider again.
- Updated `docs/architecture/backend-foundation-status.md` so it reflects the new deploy reconciler baseline and the remaining provider/verification work honestly.
