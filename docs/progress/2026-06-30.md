# 2026-06-30 Progress

## Scope

Deploy provider-operation hardening after Netlify reconciliation pattern-mining and follow-up architecture reviews, plus the first real HTTP post-deploy verifier baseline.

## Completed

- Promoted `provider_operation_status` to a contract-owned enum with `manual_reconciliation_required`.
- Split `SiteHostingPort` into phased provider operations:
  - `beginDeploy`,
  - `uploadDeployFiles`,
  - existing `getDeploy`,
  - composed `createDeploy` retained for convenience.
- Changed the Netlify adapter to:
  - send a traceable deploy title,
  - create the async digest deploy before upload,
  - poll for required file digests,
  - return an opaque resume token,
  - upload files from the supplied resume token.
- Changed the deploy worker to persist `providerDeployId` and upload resume evidence before file upload.
- Added upload-resume behavior for recorded provider deploys so a retry after a mid-upload crash checks provider state and re-uploads from persisted evidence when local upload completion is not recorded.
- Changed `in_flight` without a recorded provider deploy id from ordinary `failed` to `manual_reconciliation_required`.
- Added worker and repository guards so manual reconciliation rows are not overwritten by `startDeployment`, `markProviderMutationInFlight`, provider success/pending writes, or failure handling.
- Added ADR 0009 to record the accepted decisions and rejected ideas:
  - no automatic recreate from unknown provider state,
  - no advisory locks across provider I/O,
  - no automatic provider-resource deletion compensation,
  - no recovery-point/provider-operation table until the model needs it.
- Replaced the synthetic release `verify()` response with a `VerificationPort` backed by `HttpReleaseVerificationAdapter`.
- The verifier now checks intended live routes derived from deployment live URL + release item routes:
  - HTTP success,
  - noindex robots directives,
  - canonical URL/trailing slash match,
  - JSON-LD parseability,
  - sitemap inclusion,
  - tracking marker presence when tracking is configured.
- Release verification now persists `release_verifications` and `release_verification_checks`, then updates deployment verification status and deployment health status from the observed checks.
- Follow-up verification fixes aligned the renderer and verifier contract so rendered pages can emit approved canonical and JSON-LD fields, prefer stable production URLs over deploy permalinks, avoid a committed `verifying/running` pre-state in the inline API path, update the release plan status from verification truth, and match sitemap `<loc>` entries exactly.
- Started Foundation Milestone 3 lifecycle integration coverage:
  - added `test:integration` scripts,
  - added a disposable Postgres migration/reset harness guarded by `TEST_DATABASE_URL`,
  - added the first `ReleasesService.verify()` API/DB integration spec covering healthy verification, rollback recommendation, verifier failure evidence, release-plan status projection, and cross-project rejection.

## Verified

```text
corepack pnpm --filter @localseo/adapters test
corepack pnpm --filter @localseo/api test
corepack pnpm --filter @localseo/api typecheck
corepack pnpm --filter @localseo/worker test
corepack pnpm --filter @localseo/db db:check
corepack pnpm check
corepack pnpm --filter @localseo/api test:integration
$env:TEST_DATABASE_URL="postgres://postgres:postgres@localhost:5432/local_seo_test"; corepack pnpm --filter @localseo/api test:integration
```

`test:integration` currently skips unless `TEST_DATABASE_URL` is configured. The configured database must be disposable; the harness resets the `public` schema and refuses to run unless the database name contains `test` or `integration` or `LOCALSEO_ALLOW_TEST_DB_RESET=true` is set.

PostgreSQL 17 is now available as a user-space runtime under `%USERPROFILE%\.localseo-tools`, and the configured integration command passes against `local_seo_test`. Integration coverage details and local database commands are documented in `docs/testing/lifecycle-integration-coverage.md`.
