# Progress 2026-07-04

## Opportunity Scout Real Loop

- Started the next agent-first MVP milestone after the backend Explorer and OpenCode Go adapter baselines:
  - real-provider smoke tooling,
  - prompt/evidence packet tuning,
  - operator-owned opportunity lifecycle decisions.
- Added the smoke procedure in `docs/testing/opportunity-scout-smoke.md`:
  - a real run may end in `succeeded`, `output_not_json`, `output_schema_mismatch`, or `qa_rejected` and still count as a useful smoke when the taxonomy is correct,
  - opportunities persisted is not required for the first smoke,
  - provider secrets must stay outside the repo,
  - review artifacts go under `docs/testing/scout-runs/` without raw prompts, secrets, provider blobs, or customer data dumps.
- Added local smoke tools:
  - `tools/seed-opportunity-scout-fixture.ts` seeds the deterministic Martines-style project, website import facts, GSC row/signal, tracking event, and manual ranking proof,
  - `tools/scout-smoke.ts` enqueues through the API, polls `agent_runs`, and prints only redacted summary fields.

## Prompt And Packet Tuning

- Refactored `buildOpportunityScoutPrompt()` into named, testable sections:
  - role/boundary,
  - evidence and proof rules,
  - classification/action rules,
  - nearby Orte and corridor logic,
  - competitor containment,
  - German Local SEO calibration,
  - output format.
- Added deterministic prompt coverage so CI checks that core rules remain present:
  - GSC is internal radar,
  - `proven_win` requires row-backed ranking proof,
  - nearby corridors and German examples are in the prompt,
  - output remains strict JSON matching `OpportunityScoutOutput`.
- Kept CI model-free. Real model outputs are review artifacts, not golden tests.
- Lowered the API's default real-run request size to 6 briefs while keeping the contract cap at 12.

## Opportunity Lifecycle Decisions

- Corrected the worker persistence rule:
  - `recommendedAction` is model advice only,
  - accepted scout briefs now persist with lifecycle `status = "new"` regardless of `recommendedAction`,
  - operator decisions own transitions to `monitoring`, `held`, or `rejected`.
- Added typed lifecycle persistence:
  - `opportunity_lifecycle_status` enum: `new | monitoring | held | rejected | brief_created`,
  - `opportunities.decided_by_user_id`,
  - `opportunities.status_reason`.
- Added `PATCH /projects/:projectId/opportunities/:opportunityId/status` behind `opportunity:decide`:
  - owner/admin/editor may decide,
  - viewer remains read-only,
  - reject requires a reason,
  - reopen and moves among `new`, `monitoring`, `held`, and `rejected` are allowed,
  - `brief_created` is reserved for the future create-brief/materialize-area slice.
- Updated the architecture doc so classification and lifecycle remain separate:
  - AI classification `rejected` is a model verdict,
  - lifecycle `rejected` is a human decision.
