# Progress Log

This folder records chronological project progress.

Use this for "what changed when" and link to commits, reviews, and follow-up work. Use ADRs for "why we chose this direction."

## Difference From ADRs

```text
docs/progress/
  chronological delivery log
  what changed
  what was verified
  what remains next

docs/architecture/decisions/
  decision memory
  why a choice was made
  trade-offs and regression guards
```

## When To Update

Update the current progress file after meaningful project work:

- a pushed hardening slice
- a schema/database change
- an architecture or rules change
- a review response
- a production-readiness improvement
- a major planning milestone

Do not log every tiny edit. Commit history already covers that.
