# Stack Findings

Editable notes for stack-specific review findings that are useful but not yet stable enough to become hard rules.

Use this folder for:

- official-doc refresh notes after implementation reviews
- candidate guardrails for React, TanStack, NestJS/Fastify, OAuth, URL safety, API semantics, workers, and DB access
- broader adjacent-source scans when a category is under-specified, for example TanStack Form/Table, Nest lifecycle hooks, fetch/URL Web APIs, API versioning, or worker idempotency
- links to official docs and short summaries of why a finding matters

Do not put Local SEO product truth here. Promote only stable, recurring implementation lessons into `.ai-stack-rules/`.
